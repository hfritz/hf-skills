# HF Brand Voice Reference

Personal presentation voice for Helmut Fritz — product leader, builder, and storyteller.

## Voice in One Line

**Direct. Warm. Evidence-led.** Say the thing clearly, back it with data, and make the person in the room feel like they're talking to someone who gets it.

## Core Characteristics

### 1. Direct
*"Respect the audience's time. Say the headline first."*
- Lead with the outcome, not the build-up
- The title of each slide should be its conclusion, not a vague topic
- Cut throat-clearing — don't explain what you're about to say; say it
- If you can say it in 10 words, don't use 20

**Good:** "Revenue up 83% per order after group ordering launch"
**Avoid:** "Overview of the results we saw following the group ordering feature launch"

### 2. Warm
*"Authoritative doesn't mean cold. Smart doesn't mean distant."*
- Write like you're in the room, not submitting a report
- Use first person — "we built", "I learned", "the team found"
- Acknowledge complexity and trade-offs honestly
- Human-scale context: "15 partners across 4 markets" lands better than "n=15"

### 3. Evidence-led
*"Opinions without numbers are just opinions."*
- Anchor every claim in a metric, a quote, a user behaviour, or a decision
- Name the source: "Based on 3 months of A/B test data" or "From 20 user interviews"
- Make causality clear: what changed, why, because of what action
- When data isn't available, say so and name the assumption

### 4. Story-shaped
*"A deck is a story, not a document dump."*
- Every deck has a beginning (situation), middle (complication + response), end (outcome + learning)
- Slides should flow — each one answers a question the previous slide raised
- Tension makes decks memorable: show the gap between where you were and where you got to

## Writing Style

### Slide titles
- State the conclusion, not the topic
- Use sentence case: "Customer drop-off reduced by 30%" not "Customer Drop-Off Results"
- Max 1 punchy sentence — if it wraps to 2 lines, cut it

### Body text
- Short paragraphs or tight bullet lists (3–5 points max)
- One idea per bullet — never combine two claims with "and"
- Active voice: subject → verb → object
- No passive voice in results: "we reduced drop-off" not "drop-off was reduced"

### Numbers
- Always include the unit: "€40M GMV", "+12pp retention", "28-min delivery"
- Compare to baseline when you have one: "+57% AOV (€28 → €44)"
- Round unless precision matters: "~€100M" rather than "€97.3M" for high-level slides

### Tone calibration by audience

| Audience | Tone | Emphasis |
|----------|------|----------|
| C-suite / investors | Crisp, decisive, strategic | Outcome + scale |
| Hiring managers | Confident, specific, reflective | Your contribution + learning |
| Cross-functional peers | Collaborative, honest, practical | Trade-offs + how we got there |
| Design / engineering | Precise, curious, direct | Problem framing + decisions |

## Things to Avoid

| Avoid | Use instead |
|-------|------------|
| "This deck will cover…" | Just start with the thing |
| "As you can see…" | Trust the slide |
| "Very", "extremely", "significantly" | Give the number |
| "Learnings" | "What we learned" |
| "Synergies", "leverage", "scalable" | Say the actual thing |
| "We feel that…" | "The data shows…" or just assert it |
| Passive voice in headlines | Active: "we did X", not "X was done" |
| Title Case headlines | Sentence case always |

## Slide Copy Length Guidelines

| Element | Target length | Never exceed |
|---------|--------------|-------------|
| Slide title | 6–10 words | 14 words |
| Bullet point | 8–15 words | 20 words |
| Body paragraph | 40–80 chars | 150 chars per text block |
| Quote / verbatim | 80–140 chars | 200 chars |
| Stat callout label | 3–6 words | 8 words |
| Footer / attribution | 1–2 short lines | 2 lines |

## Storytelling Patterns That Work

### The gap pattern (for case studies)
> "The situation was X. The problem was Y. We did Z. The result was [number]."

### The opportunity pattern (for pitches)
> "Most people think A. But actually B. Which means we can C."

### The learning pattern (for retrospectives)
> "We expected X. We got Y. The reason was Z. Next time we'd do W."

### The ask pattern (for approvals / resources)
> "We're here, we want to get here, the gap is X. To close it we need Y."
